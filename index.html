      • ¿Qué es una Aplicación Web Progresiva?

      Las aplicaciones web progresivas (PWA) son aplicaciones creadas utilizando tecnologías web que se pueden instalar y ejecutar en todos los dispositivos según el código. Las aplicaciones web progresivas (PWA) se crean y amplían con API modernas para proporcionar mayor capacidad, confiabilidad e instalación, haciéndolas accesibles a cualquier persona, en cualquier lugar y en cualquier dispositivo con una única base de código.

      ¿Cómo se crean?

      Las aplicaciones web progresivas se escriben utilizando los lenguajes de programación de Internet: HTML, CSS y JavaScript y se distribuyen a los usuarios a través de servidores web. Para que la aplicación esté disponible para los usuarios, se debe de alojar en un servidor web al que se pueda acceder a través de HTTPS. El servidor debe de contener

    • Código de back-end:

      • Los puntos de conexión necesarios que la aplicación necesita, cuando está conectada a Internet, para recuperar contenido dinámico que se puede almacenar en una base de datos en el servidor.

    • Código front-end

      • Los recursos necesarios para instalar la aplicación en el dispositivo del usuario, p. B. Código HTML, CSS y JavaScript

      Ventajas

    • Almacenamiento en caché para almacenar recursos de aplicaciones estáticas.
    • Almacenamiento local y de sesión, para almacenar pequeñas cantidades de datos del usuario.
    • Redirigir a páginas personalizadas sin conexión para páginas no almacenadas en caché.
    • Almacenamiento en caché de todos los recursos estáticos que su aplicación necesita para iniciarse, representar contenido y permitir a los usuarios realizar tareas comunes.
    • Manejo adecuadamente de las tareas de la aplicación que no se pueden utilizar sin acceso a Internet.
    • Implementa escenarios avanzados, como permitir a los usuarios descargar datos para verlos sin conexión.

      • Capacidad

      En una aplicación web progresiva la capacidad amplía lo que la web puede hacer con funciones como acceso al sistema de archivos, controles multimedia, credenciales de aplicaciones y compatibilidad total con el portapapeles. Todas estas capacidades se basan en un modelo de permisos web seguro y centrado en el usuario, lo que garantiza que visitar sitios web no sea una tarea desalentadora para los usuarios.

      Las PWAs son altamente personalizables y pueden adaptarse para satisfacer las necesidades específicas de un proyecto o aplicación.

      compatibilidad

      La compatibilidad en las PWA es esencial para garantizar que los usuarios puedan acceder y disfrutar de la aplicación en una variedad de entornos y dispositivos.

      Cuando se está conectado a una red lenta o poco confiable o incluso sin acceso a Internet, las aplicaciones instaladas se pueden abrir y utilizar normalmente.

      Los usuarios de aplicaciones instaladas esperan que sigan funcionando en estas condiciones. También esperan que las partes de la aplicación que dependen de la red manejen adecuadamente las desconexiones con mensajes personalizados y capacidades de almacenamiento en caché local.

      Integración en el sistema operativo

      Los usuarios esperamos que las aplicaciones instaladas se integren de alguna manera con el sistema operativo. La aplicación debe poder proporcionar una experiencia más familiar, atractiva y fluida al aprovechar las funciones de integración del sistema operativo, como accesos directos, uso compartido entre aplicaciones, control de errores, archivos o enlaces.
